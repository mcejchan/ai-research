## Key Takeaways

### 1. AMP as a Coding Agent
**Quote:** "AMP is an opinionated frontier agent."  
**Why it matters:** AMP represents a shift in how coding agents operate, focusing on a unique architecture that emphasizes user experience and efficiency. By embracing the evolving landscape of AI in coding, AMP aims to redefine the developer's interaction with code.

### 2. Custom Tool Set Over MCP
**Quote:** "We should really actually focus most of our attention on the core set of tools within AMP."  
**Why it matters:** AMP prioritizes a refined custom tool set over generic MCP integrations to enhance agent performance. This decision allows for better context management and reduces confusion, ultimately leading to more effective coding outcomes.

### 3. Sub Agents for Context Management
**Quote:** "Sub agents are effectively a way to conserve and extend the context window of your main agent."  
**Why it matters:** The introduction of sub agents allows AMP to manage context more efficiently, enabling the main agent to focus on relevant tasks without being bogged down by excessive information. This innovation enhances the overall functionality of the coding agent.

## Practical Tips & Procedures

### Efficient Code Review Process
**What it does:** Streamlines the code review process when using AMP.  
**Steps:**
1. Use the AMP agent panel to drive edits rather than manually editing code.
2. Utilize the custom diff viewer to review changes, focusing on the guided tour of changes.
3. Engage with the agent to clarify any complex edits or issues that arise during the review.
**Quote:** "We actually built a custom diff viewer for the way that people are consuming agentic output."  
**When to use:** When reviewing large changes or collaborating with team members using AMP.
