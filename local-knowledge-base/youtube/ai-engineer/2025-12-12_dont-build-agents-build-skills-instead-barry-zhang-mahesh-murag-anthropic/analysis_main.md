# Don't Build Agents, Build Skills Instead – Barry Zhang & Mahesh Murag, Anthropic

## Key Takeaways

### 1. Shift from Agents to Skills
**Quote:** "We stopped building agents and started building skills instead."  
**Why it matters:** This shift emphasizes the importance of domain expertise over general intelligence in agents. Skills provide a structured way to package procedural knowledge, making agents more effective in specific tasks.

### 2. Skills as Composable Knowledge
**Quote:** "Skills are organized collections of files that package composable procedural knowledge for agents."  
**Why it matters:** This organization allows for easy creation and sharing of skills, enabling both technical and non-technical users to enhance agent capabilities without extensive coding knowledge.

### 3. The Role of Domain Expertise
**Quote:** "Agents today are a lot like Mahesh. They're brilliant, but they lack expertise."  
**Why it matters:** Highlighting the need for domain-specific knowledge ensures that agents perform tasks accurately and efficiently, rather than relying solely on their general capabilities.

## Practical Tips & Procedures

### Creating and Using Skills
**What it does:** Provides a framework for building and implementing skills for agents.  
**Steps:**
1. Organize procedural knowledge into folders with clear instructions.
2. Use markdown files (skill.md) to document skills and their usage.
3. Share skills via platforms like Git or Google Drive for team collaboration.  
**Quote:** "These also work with what you already have."  
**When to use:** When developing new capabilities for agents in specific domains or tasks.

## Tools & Technologies Mentioned
- **Cloud Code:** A coding agent that allows for general-purpose automation and task execution.

## Important Warnings/Risks
- **Lack of Expertise:** "They can do amazing things when you really put in the effort and give proper guidance, but they're often missing the important context up front." - This highlights the risk of relying on agents without adequate domain knowledge, which can lead to errors in critical tasks.
