## Practical Tips & Procedures

### Setting Up Cloud MD Files
**What it does:** Organizes project documentation for Claude Code.
**Steps:**
1. Run the `/init` command to generate an overview of your codebase.
2. Create a cloud MD file for each subfolder to track functions and file structures.
3. Maintain a change log to document updates and rationale.
**Quote:** "If you have a good change log, it's easy for cloud to realize what were the changes that were made and why."
**When to use:** Use this process for every new project to ensure Claude Code has the necessary context.

### Automating Code Reviews
**What it does:** Streamlines the code review process.
**Steps:**
1. Use built-in commands like "review PR comments" to automate fetching comments.
2. Tag Claude in PRs to initiate review tasks.
3. Generate PRs for features by listing them and tagging Claude.
**Quote:** "I can do way more things using this process than I can just on my own."
**When to use:** Implement this when managing multiple PRs or features to save time and ensure thorough reviews.
