**Quote:** "Skills went from personal configuration 6 months ago to organizational infrastructure today... Now, team and enterprise admins are rolling out skills workplacewide."
**Why it matters:** Skills are no longer just personal tools but are now being integrated into organizational workflows. ### 2. **Agent-First Skills**
**Quote:** "Humans were the caller of skills in October... Now the majority of skills are called by agents because agents can make hundreds of skill calls over the course of a single run."
**Why it matters:** The shift to agent-first skills highlights the need for designing skills that are optimized for machine use. Agents can execute tasks at scale, making skills a critical tool for automating complex workflows. This requires precise, well-tested skills to avoid costly errors.

### 3. **Skills as a Cross-Industry Standard**
**Quote:** "Skills is now an open standard... Fundamentally, you need to start thinking of skills as a common infrastructure vehicle."

### 4. **Compounding Value of Skills**
**Quote:** "Skills compound by the weight of industry investment in the ecosystem and by the weight of your own commitment to having a predictable pattern for doing something and writing it down."
**Why it matters:** Unlike prompts, which are ephemeral, skills are reusable and improve over time. This compounding effect makes them a valuable asset for both individuals and organizations, enabling continuous improvement and scalability.

### 5. **Testing and Refining Skills for Agents**
**Quote:** "You need to have a test suite that you run against your skill... Skills don't always change in predictable ways."
**Why it matters:** As agents increasingly rely on skills, rigorous testing is essential to ensure reliability. Poorly designed or untested skills can lead to failures, especially in agent-driven workflows. Quantitative testing and refinement are critical for maintaining performance.

# Practical Tips & Procedures

### **Building Effective Skills**
**What it does:** Ensures skills are clear, reliable, and optimized for both agents and humans.  
**Steps:**
1. **Write a clear description:** Avoid vagueness. Include specific trigger phrases, document types, and expected outputs.  
2. **Define methodology:** Include reasoning, frameworks, and principles, not just linear steps.  
3. **Specify output format:** Clearly state the desired format (e.g., Markdown, Excel, PDF).  
4. **Include edge cases:** Document exceptions and scenarios to guide the AI.  
5. **Provide examples:** Add sample outputs to help the AI understand what "good" looks like.  
**Quote:** "The description is where most skills go to die... A good description names the document types or the artifact types it produces."  
**When to use:** When creating new skills or refining existing ones for better performance.

---

### **Testing Skills for Agent Use**
**What it does:** Ensures skills are reliable and effective when called by agents.  
**Steps:**
1. Develop a test suite to evaluate skill performance.  
2. Quantify results and identify areas for improvement.  
3. Refine the skill, update the version, and retest.  
**Quote:** "You need to run a basket of tests, quantify the results, change the skill like a version number, and then come back and see if it does a better job."  
**When to use:** Before deploying skills in agent-driven workflows.

---

### **Structuring Skills for Teams**
**What it does:** Organizes skills for different levels of use within an organization.  
**Steps:**
1. **Tier 1 - Standard Skills:** General skills like brand voice or formatting rules, rolled out organization-wide.  
2. **Tier 2 - Methodology Skills:** High-value workflows specific to teams or senior practitioners.  
3. **Tier 3 - Personal Workflow Skills:** Individualized tools that can be shared when needed.  
**Quote:** "Think of tier 2 skills as what are the things you would want to communicate to a new hire that would take them months to learn otherwise."  
**When to use:** When organizing skills for team or enterprise use.
